# AnnotationCheck — 标注审核工具技术设计文档

> 版本 v0.2 | 2026-04-14

---

## 1. 项目背景与目标

### 1.1 数据集结构

```
data/
├── visual/
│   └── {seq_name}/
│       └── {seq_name}/
│           ├── visible/      # 可见光帧：v000001.jpg, v000002.jpg ...
│           └── infrared/     # 红外帧：  i000001.jpg, i000002.jpg ...
└── text/
    └── {seq_name}.txt        # 文本标注：每行对应一帧，行号 = 帧号（Unix LF，末行无换行）
```

**已确认的数据特性：**
- 可见光与红外帧严格一一对应（文件名序号相同，帧数完全一致）
- 文本行数 = 图像帧数（使用 `splitlines()` 解析，非 `wc -l`）
- 文件编码：UTF-8，换行符：Unix `\n`，末行无末尾 `\n`
- 当前标注平均长度约 18 词（12~32 词），字符约 106（63~187）

### 1.2 核心目标

构建一个桌面 GUI 工具，让标注审核员能够：
- 同步浏览可见光与红外双模态图像序列
- 对照图像逐帧审查对应文本描述，依据规范完成四类修正
- 实时获得规范违规提示（长度、重复、语言混杂等），辅助高效审核
- 在界面内直接编辑、保存标注，留存完整备份
- 标记问题帧并追踪审核进度

## 2. 标注规范（来源：LasHeR 数据集文本描述人工校验说明文档）

审核工具的所有校验逻辑以本节为依据。

### 2.1 长度规范

| 级别 | 词数 | 处理方式 |
|------|------|---------|
| 正常 | ≤ 20 词 | 无提示 |
| 警告 | 21 ~ 30 词 | 橙色警告，建议精简 |
| 违规 | > 30 词 | 红色错误，必须修改 |

> 精简原则：删除冗余形容词和修饰语，保留核心视觉信息。

### 2.2 相邻帧多样性规范

- **严禁**相邻帧（前一帧或后一帧）之间出现完全相同的文本
- **强烈不建议**相邻帧之间文本高度相似（相似度 > 90%）
- 目的：提升模型对语言变体的鲁棒性，避免过拟合

### 2.3 四类必须修正的错误

| 类型 | 说明 | 检测关键词 |
|------|------|-----------|
| **幻觉（HALLUCINATION）** | 描述图像中不存在的内容（虚构颜色、标识、物体类型） | 需人工判断，不可自动检测 |
| **语法错误（GRAMMAR）** | 拼写错误、时态不一致、冠词误用、主谓不匹配 | 需人工判断 |
| **语言混杂（MIXED_LANG）** | 含中文字符、乱码符号（如 `、` `等`） | 可自动检测 |
| **视觉不准确（VISUAL_INACCURACY）** | 描述未对应绿框内目标，或颜色/形状/动作错误 | 需人工判断 |

### 2.4 特殊情况处理

| 情况 | 规范做法 |
|------|---------|
| 目标模糊难辨 | 查看前后帧推断，结合上下文描述可见部分 |
| 目标被遮挡 | 仅描述可见部分，不推测遮挡部分 |
| 目标消失 | 填写标准文本：`No visible target in current frame` |

### 2.5 AI 工具使用限制

- **严禁**批量使用大模型对图像和文本进行自动校验
- **允许**对单条难以表达的描述使用大模型润色，但必须人工审核输出
- **底线**：每条最终提交的文本必须经过人工确认

## 3. 技术选型

| 层级 | 选型 | 理由 |
|------|------|------|
| GUI 框架 | PyQt6 / PySide6 | 成熟、跨平台、控件丰富；支持自定义布局与键盘事件 |
| 图像处理 | Pillow + QPixmap | Pillow 读图，Qt 渲染，避免引入 OpenCV 依赖 |
| 文本编辑 | 自定义 QPlainTextEdit | 支持行号边栏、实时高亮、违规标注 |
| 相似度计算 | `difflib.SequenceMatcher` | 标准库，用于相邻帧重复检测，无需额外依赖 |
| 语言检测 | 正则表达式（`\u4e00-\u9fff`） | 轻量检测中文字符，无需 langdetect 库 |
| 配置持久化 | JSON（`config.json`） | 保存上次路径、偏好布局等会话状态 |
| 备份策略 | 文件系统（带时间戳） | 简单可靠，无需数据库 |

## 4. 功能模块详细设计

### 4.1 序列加载模块

**入口**：工具栏「打开数据集」按钮，弹出目录选择对话框（选择 `data/` 根目录）。

**加载逻辑**：
1. 扫描 `data/visual/` 下所有子目录，构建序列列表。
2. 用户选择序列后，自动推断路径：
   - 可见光：`data/visual/{seq}/{seq}/visible/v*.jpg`（`natsort` 自然排序）
   - 红外：`data/visual/{seq}/{seq}/infrared/i*.jpg`
   - 文本：`data/text/{seq}.txt`（用 `splitlines()` 解析，保留原始 `\n` 格式写回）
3. 若文本文件不存在，弹出警告并提示联系负责人（规范 4.1 第 3 条）。
4. 加载完成后状态栏显示：序列名、帧总数、文本行数、行帧差。

**加载时校验**（规范 4.1）：
- 可见光帧数 ≠ 红外帧数 → 弹出错误对话框，提示联系负责人
- 文本行数 ≠ 图像帧数 → 状态栏红色警告：`⚠ 行帧不一致：文本 112 行 / 图像 113 帧`
- 对全部文本行执行一次批量违规扫描（长度、语言混杂、完全重复），结果缓存用于初始高亮

### 4.2 序列浏览面板

提供**可滚动的序列列表**，自动扫描 `data/visual/` 下所有序列：

```
┌─ 序列列表 ─────────────────┐

- 违规数徽章：序列加载时批量扫描可检测项（长度 >30 词、语言混杂），数量显示在右侧
- 双击或回车切换序列，若当前有未保存修改弹出确认对话框

### 4.3 图像显示模块

#### 4.3.1 双模态窗格布局

默认布局（**模式 A**，可见光为主）：

```
┌─────────────────┬──────────────────────┐
│                 │                      │
│   可见光 (大)    │   文本标注窗格         │
│                 │                      │
├─────────────────┤                      │
│  红外光 (小)    │                      │
└─────────────────┴──────────────────────┘
```

切换后布局（**模式 B**，红外光为主）：

```
┌─────────────────┬──────────────────────┐
│                 │                      │
│   红外光 (大)    │   文本标注窗格         │
│                 │                      │
├─────────────────┤                      │
│  可见光 (小)    │                      │

- 两个图像窗格使用 `QSplitter`（垂直）实现，可自由拖拽调节比例，默认 7:3
- 「切换主模态」按钮（工具栏 + 快捷键 `Tab`）一键交换位置与尺寸记忆
- 图像在窗格内**保持宽高比**缩放，窗格 `resizeEvent` 时自动重绘

#### 4.3.2 图像渲染

- 图像等比缩放居中，背景填黑（`Qt.SmoothTransformation`）
- 左下角叠加显示帧号徽章（`QPainter.drawText`，半透明背景）
- 当前帧有违规时图像窗格边框变色：橙色=警告，红色=错误
- 支持**鼠标滚轮缩放**（`Ctrl + 滚轮`）+ 拖拽平移（按住左键）
- 双击图像弹出**全屏预览**（`QDialog` 全屏，支持左右键翻页）

### 4.5 文本标注模块

#### 4.5.1 文本窗格布局

```
┌─ 文本标注 ─────────────────────────────────────────────────┐
│ 行号 │ 状态 │ 标注内容                              │词数  │
│  45  │  ✓   │ A person in white walks near a car.   │ 8w  │ ← 正常
│  46  │  🟠  │ A person in a white top and dark ...  │ 22w │ ← 长度警告（橙）
│▶ 47  │  🔴  │ The bus is large rectangular vehicle  │ 31w │ ← 超出上限（红，当前帧）
│  48  │  🔁  │ A person in white walks near a car.   │ 8w  │ ← 与47行重复（蓝）
│  49  │  🈶  │ 一个穿白色衣服的人 walks near a car.  │ 9w  │ ← 语言混杂（紫）
└────────────────────────────────────────────────────────────┘
[🔍 搜索]  [全局替换]  [跳转当前帧]  [违规: 3 处 ▼]
```

**违规颜色编码（行背景色）**：

| 颜色 | 含义 | 对应规范 |
|------|------|---------|
| 橙色背景 | 长度警告（21~30 词） | 规范 2.1 |
| 红色背景 | 超出绝对上限（>30 词） | 规范 2.1 |
| 蓝色背景 | 与相邻帧完全重复 | 规范 2.2 |
| 黄色背景 | 与相邻帧高度相似（>90%） | 规范 2.2 |
| 紫色背景 | 含中文字符或乱码符号 | 规范 2.3 |
| 灰色背景 | 行数多于图像帧（冗余行） | 规范 4.1 |

**当前帧行**：在所有背景色基础上叠加加粗边框，用于区分高亮。

#### 4.5.2 实时校验引擎（`annotation_validator.py`）

每次文本内容变化时（编辑或加载）触发校验，返回每行的违规列表：

```python
class Violation:
    frame_idx: int
    type: str          # 'OVER_LIMIT' | 'OVER_WARN' | 'DUPLICATE' | 'SIMILAR' | 'MIXED_LANG'
    severity: str      # 'error' | 'warning' | 'info'
    detail: str        # 人读描述，如 "31 words (limit: 30)"

class AnnotationValidator:
    def validate_all(self, lines: list[str]) -> dict[int, list[Violation]]: ...
    def validate_line(self, idx: int, lines: list[str]) -> list[Violation]: ...
```

**各项校验实现**：

1. **长度校验**：`len(line.split()) > 30` → OVER_LIMIT；`> 20` → OVER_WARN
2. **重复检测**：与 `lines[idx-1]` 和 `lines[idx+1]` 逐字比较，完全相同 → DUPLICATE
3. **相似检测**：`difflib.SequenceMatcher(None, a, b).ratio() > 0.9` → SIMILAR
4. **语言检测**：`re.search(r'[\u4e00-\u9fff\uff00-\uffef]', line)` → MIXED_LANG（覆盖中文、全角符号）
5. **乱码检测**：`re.search(r'[\x00-\x08\x0b\x0c\x0e-\x1f\x7f-\x9f]', line)` → MIXED_LANG

> **性能**：全量序列（最长约 300 帧）验证在主线程中完成，约 < 5ms，无需异步。

#### 4.5.3 词数实时显示

文本窗格右侧每行显示词数徽章，进入编辑模式时当前行词数**实时更新**，超过 20 词变橙，超过 30 词变红。

#### 4.5.4 编辑功能

- 文本窗格默认**只读**模式；点击「编辑」按钮或双击当前帧行进入编辑模式
- 编辑模式下状态栏显示：`编辑中 — 未保存 | 当前行: 22 词`
- 支持：
  - 修改当前行内容（编辑完成后自动触发单行重新校验）
  - 插入行：`Ctrl+Enter`（当前行后）/ `Ctrl+Shift+Enter`（当前行前）
  - 删除当前行：`Ctrl+Delete`（弹出确认，因为会导致行帧错位）
  - 全局查找：`Ctrl+F`
  - 全局查找替换：`Ctrl+H`（支持正则，用于批量修正重复表达）

**删除行的特殊处理**：删除任意行会导致该行之后所有帧的文本与图像错位，操作前弹出警告：`"删除第 N 行将导致此行之后所有标注错位，确认继续？"`

#### 4.5.5 保存与备份

**保存逻辑**（`Ctrl+S`）：
1. 将当前编辑内容以 `\n` 连接（末行不加 `\n`）写回 `data/text/{seq_name}.txt`
2. 同时在 `data/text/backup/` 下保存带时间戳副本：
   ```
   data/text/backup/{seq_name}/{seq_name}_20260414_153022.txt
   ```
3. 每个序列最多保留最近 **10 个**备份版本（超出时删除最旧的）
4. 保存成功后触发全量重新校验，状态栏显示「已保存 ✓」持续 3 秒

**自动保存**：
- 每 3 分钟若有未保存修改则静默自动保存（不触发备份，仅覆写主文件）
- 切换序列时若有未保存修改，弹出对话框：「保存 / 放弃 / 取消」

### 4.6 问题帧标记模块

按 `F` 键或工具栏按钮标记当前帧，支持**区分问题类型**：

```
标记当前帧问题类型：
  ○ 幻觉 (HALLUCINATION)     ← 描述了图像中不存在的内容
  ○ 语法错误 (GRAMMAR)        ← 拼写/时态/冠词等语法问题
  ○ 视觉不准确 (VISUAL)       ← 描述未对应绿框内目标
  ○ 其他 (OTHER)
[可选备注输入框]  [确认]  [取消]
```

问题帧信息存储在 `data/review/{seq_name}_flags.json`：
```json
{
  "flagged_frames": {
    "3": {"type": "HALLUCINATION", "note": "描述了红色车辆，图中实为蓝色"},
    "17": {"type": "VISUAL", "note": "描述了框外背景人物"},
    "42": {"type": "GRAMMAR", "note": "冠词 a/an 使用错误"}
  }
}
```

**自动违规帧**（无需手动标记，由校验引擎自动生成）：
- OVER_LIMIT、MIXED_LANG 等可检测违规直接在违规帧列表中显示，与手动问题帧分开展示

**问题帧面板**（点击工具栏「问题帧」展开）：
```
┌─ 问题帧列表 ──────────────────┐
│ 手动标记                       │
│  帧 3  [幻觉]  描述了红色车辆… │ ← 点击跳转
│  帧17  [视觉]  描述了框外…    │
│ 自动检测                       │
│  帧46  [长度]  22词 超过建议值  │
│  帧49  [混语]  含中文字符       │
└───────────────────────────────┘
[导出报告]
```

### 4.7 审核进度模块

审核状态存储在 `data/review/progress.json`：
```json
{
  "bus": {
    "status": "done",
    "last_frame": 113,
    "reviewed_at": "2026-04-14",
    "violation_count": 0,
    "manual_flag_count": 2
  },
  "blackbagbike": {
    "status": "in_progress",
    "last_frame": 47,
    "violation_count": 3
  }
}
```

- 序列浏览面板颜色：灰=未开始，黄=进行中，绿=已完成，红角标=有自动检测违规
- 标记「已完成」前检查是否还有未处理的 OVER_LIMIT / MIXED_LANG 违规，有则提示
- 退出时自动保存当前帧号，下次从该帧继续

## 7. 关键实现要点

### 7.1 图像懒加载与预读

- 不在初始化时加载全部帧，按需读取当前帧
- 使用 `QThreadPool` + `QRunnable` 在后台预加载 ±3 帧，结果缓存在 `dict[int, QPixmap]` 中
- 切帧时优先从缓存取，缓存未命中时同步加载（通常 < 20ms）
- 缓存上限 20 帧，LRU 淘汰

### 7.2 撤销/重做

- `annotation_manager.py` 维护文本快照列表（`list[list[str]]`），每次编辑前 push 副本
- 默认保留最多 50 步历史；`Ctrl+Z` 弹出快照，`Ctrl+Y` 重新应用
- 撤销/重做后触发全量重新校验并刷新高亮

### 7.3 行帧双向联动

- 切换帧 → 文本窗格滚动到对应行并高亮当前帧边框
- 单击文本行 → 图像跳转到对应帧
- 用 `_syncing: bool` 标志位防止信号循环触发

### 7.4 校验结果缓存与增量更新

- 加载序列时执行全量校验，结果存入 `dict[int, list[Violation]]`
- 编辑单行时仅重新校验该行及相邻两行（因重复/相似检测跨行）
- 校验结果变化时通知文本窗格和进度面板更新显示

### 7.5 高 DPI 支持

```python
QApplication.setHighDpiScaleFactorRoundingPolicy(
    Qt.HighDpiScaleFactorRoundingPolicy.PassThrough
)
```
图像渲染使用 `Qt.SmoothTransformation`，图标使用 SVG 格式。

### 7.6 写回文件格式保持

```python
# 读取
with open(path, 'r', encoding='utf-8') as f:
    lines = f.read().splitlines()   # 正确：不含末行换行符

# 写回
with open(path, 'w', encoding='utf-8', newline='\n') as f:
    f.write('\n'.join(lines))       # 无末尾换行，与原格式完全一致
```

## 8. 已确认事项

| # | 问题 | 确认结论 |
|---|------|---------|
| 1 | 换行格式 | Unix `LF`（`\n`），末行无换行符，写回用 `'\n'.join(lines)` |
| 2 | 可见光与红外帧对应关系 | 严格同帧，帧数完全一致（已抽查 4 个序列） |
| 3 | 文本行数与帧数关系 | 完全一致，`wc -l` 少报 1 是末行无 `\n` 所致 |
| 4 | 备份版本数 | 每序列最多保留最近 10 份 |
| 5 | 多人协作 | 单人使用，无需锁文件 |
| 6 | 标注格式校验 | 已明确：长度（≤20/≤30词）、相邻重复、语言混杂 |
| 7 | 交付形式 | 先以 Python 脚本形式交付，后续可考虑 PyInstaller 打包 |
